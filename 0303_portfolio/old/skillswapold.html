Skill Swap.

A mobile application where students can share, learn, and grow through skill exchange.

February - June 2025 International Team Group Project
Project Goal

Why SkillSwap was created.

Students often want to learn new skills but lack accessible ways to exchange knowledge with peers. SkillSwap provides a platform that enables collaborative learning and skill sharing among students.

Problem Statement

Understanding the learning gap.

  • Void of free skill learning platforms
  • Void for peer to peer + community driven learning
  • Untapped knowledge within student community
Proposed Solution

How SkillSwap solves the problem.

SkillSwap App allows students to offer skills, collect credits, and spend them on learning.

  • Collaboration spaces for group projects
  • Smart matching for best peer-to-peer connections
  • A non-monetary credit system for fair exchange
  • Safety & transparency via university email login and feedback system

Business Model

How the platform sustains itself.

  • Free Version

  • project participation limit (1 per month)
Credit System

How skill exchange works.

Value of the non-monetary Credits
  • measurement of time + effort put into skills exchanges
  • Complexity of Skills (more credits for more advanced skills)
  • set by each user themselves (after suggested value of a skill)
Credit Calculation
  • Standard credit rate suggestions (based on basic - advanced skills, different skill levels)
  • users can use those or adjust them acc. to their estimation
  • personal negotiations possible (both parties have to agree!)
Earning Credits - Teaching
  • Offering to teach a skill => earning credits
  • Depending on what they teach, they earn a fixed amount of credits
Learning Credits - Spending / Using
  • Users spend credits to learn/use skills
  • session is confirmed => credit taken out
  • automatic amount check - only options that user has enough credits for are possible + shown
  • 1. Video call validated by both sides => credits taken out
  • automatic amount check - no exchanges without enough credits
  • option to spend in voucher store on collaboration partners too
Verfication
  • session confirmed = credits taken from user A
  • both users confirm the successfully completed session = credit is added to user B
