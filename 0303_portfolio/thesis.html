Overview

Security awareness exists.
Engagement doesn't.

Organisations invest heavily in IT security training — but employees don't attend, ignore the emails, and forget what they learned. The problem isn't the content. It's the format. Traditional training demands active participation from people who are already context-switching all day.

But every employee walks past screens every single day. Public displays offer high visibility, repeated exposure, and zero entry barrier. This project explores how to turn those brief, incidental moments into meaningful, interactive learning experiences — without asking for commitment.

Work in progress. This thesis is currently in its prototyping phase. The research foundation is complete and a first lo-fi prototype has been built. Real-environment testing and formal validation have not yet been conducted.

Research Question

"How can an AI-based content framework be designed to enable meaningful security awareness learning on public displays — within a 20-second, zero-commitment interaction window?"

Core Research Insight

The problem isn't that employees don't care about security — it's that existing formats demand effort from people who are already distracted. The framework must work with the grain of natural attention patterns, not against them. Users don't want to learn. They want to react.

The Concept

Narrative micro-learning through
AI-generated decisions.

Instead of designing static content, the framework generates interactive security narratives dynamically. Users enter at any moment — each interaction is part of a 4-step story where every scene ends with a binary decision, and every path still delivers the key learning.

01

Opportunistic Engagement

The system is designed for users in motion. Each narrative begins with a recognisable trigger — a suspicious email, an unusual login, a system warning. No explanation needed. Instant context, instant curiosity. Interaction is optional; value is not.

02

Narrative Decision System

A 4-step branching story where every scene ends with a binary choice. No onboarding, no login, no penalty for leaving. Users can join or exit at any moment — but every path ensures all key content is experienced. Learning through consequence, not instruction.

03

AI Content Scalability

Instead of hand-crafting each scenario, the framework defines modular story structures, controlled branching logic, fixed tone and perspective, and reusable event templates — enabling AI to generate consistent, coherent narratives at scale across topics.

First Prototype

Early-stage, framework-driven design.

Early prototype — no formal validation yet. The current lo-fi prototype translates the narrative decision framework into a first interface concept for public displays. Informal walkthroughs have been conducted, but structured usability testing and real-environment deployment are planned for the next phase.

Trigger Scene
Opening moment — a recognisable security scenario surfaces on the display
Decision Moment
Binary choice screen — immediate, low-effort interaction without commitment
Outcome
Consequence revealed — learning delivered through narrative outcome, not instruction
What the prototype explores
  • Whether trigger scenarios create instant recognition without prior context
  • How employees respond to binary decisions on a passive public display
  • Whether a 4-step narrative delivers a learning moment within the 20-second window
  • If users can meaningfully join or exit the loop at any point
Current limitations
  • No real-environment deployment — tested in controlled walkthrough sessions only
  • AI content is simulated, not generated by a live model
  • No engagement time or interaction completion data collected yet
  • Framework not yet validated against security awareness domain criteria
